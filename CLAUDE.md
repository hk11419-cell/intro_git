## よく使うGitコマンド

これはGit学習用リポジトリなので、以下のコマンドがよく使用されます：

```bash
# 現在の状態を表示
git status

# 変更をステージング
git add first.txt
# またはすべての変更をステージング
git add .

# 変更をコミット（日本語メッセージ推奨）
git commit -m "コミットメッセージ"

# コミット履歴を表示
git log
git log --oneline
git log --all --oneline --graph  # ブランチ含む全履歴

# リモートにプッシュ
git push origin master

# リモートからプル
git pull origin master

# 差分を表示
git diff                    # ステージされていない変更
git diff --staged           # ステージされた変更
git diff HEAD               # すべての変更

# ブランチ操作
git branch -a               # 全ブランチ一覧
git checkout -b <branch>    # 新規ブランチ作成・切り替え
git switch <branch>         # ブランチ切り替え
```

## 開発上の規約

### コミットメッセージ
- 日本語で記述することを推奨（リポジトリオーナーの慣習に合わせる）
- 簡潔で何をしたか分かる内容にする
- 例: `first.txtにコンテンツを追加`, `ステージング練習`, `ブランチ操作の確認`

### ファイル編集
- このリポジトリにはビルド・テスト・lintのプロセスは存在しない
- `first.txt` を自由に編集してGit操作を練習できる
- 不要なファイルを増やさない（練習に必要なファイルのみ作成する）

### AI向け注意事項
- このリポジトリにはソースコードが存在しないため、コード分析・リファクタリング等は対象外

- コミットメッセージは日本語を優先して使用する
- `master` ブランチへの直接pushは慎重に行う
